# PhishLogic Admin Backend - Setup & Testing Guide

## Overview

Complete implementation of Phase 1 Admin Backend with authentication, all controllers, and routes.

## What's Been Implemented

### 1. Database Layer ✅
- **PostgreSQL connection pool** (`src/infrastructure/database/client.ts`)
- **Schema migration** (`src/infrastructure/database/migrations/001_initial_schema.sql`)
  - 9 tables with soft deletes and tenant_id (multi-tenant ready)
  - Default admin user (username: `admin`, password: `Admin@123`)

### 2. Authentication ✅
- **Two-tier auth model**:
  - Admin: Username/password → JWT token
  - Users: API keys (generated by admin)
- **Controllers** (`src/api/controllers/auth.controller.ts`):
  - `POST /api/auth/login/admin` - Admin login
  - `POST /api/auth/login/user` - User login with API key
  - `POST /api/admin/keys` - Create API key (admin only)
  - `GET /api/admin/keys` - List API keys (admin only)
  - `DELETE /api/admin/keys/:id` - Revoke API key (admin only)
  - `POST /api/auth/verify` - Verify current token
- **Middleware** (`src/api/middleware/auth.middleware.ts`):
  - `authMiddleware` - Validates JWT or API key
  - `requireAdmin` - Enforces admin role
  - `optionalAuth` - Injects user if authenticated

### 3. Admin Controllers ✅

- `GET /api/admin/tasks` - List all task configs

#### Debug Interface (`src/api/controllers/admin/debug.controller.ts`)

#### Notification Config (`src/api/controllers/admin/notification.controller.ts`)

- `POST /api/admin/notifications` - Create notification

#### System Settings (`src/api/controllers/admin/system-settings.controller.ts`)

## Setup Instructions

### 1. Install PostgreSQL

#### macOS (Homebrew)
```bash
brew install postgresql@15
brew services start postgresql@15
```

#### Ubuntu/Debian
```bash
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

#### Windows
Download and install from: https://www.postgresql.org/download/windows/

### 2. Create Database and User

```bash
# Connect as postgres user
sudo -u postgres psql

# In PostgreSQL shell:
CREATE DATABASE phishlogic;
CREATE USER phishlogic WITH PASSWORD 'phishlogic_dev_password';
GRANT ALL PRIVILEGES ON DATABASE phishlogic TO phishlogic;
\q
```

### 3. Run Database Migration

```bash
# Connect to database
psql -U phishlogic -d phishlogic -h localhost

# Run migration (paste contents of file)
\i src/infrastructure/database/migrations/001_initial_schema.sql

# Verify tables
\dt

# Check admin user (should see hashed password)
SELECT username, email, role, is_active, created_at FROM admin_users;

# Exit
\q
```

### 4. Install Dependencies

```bash
npm install
```

All required dependencies are already in `package.json`:
- `pg` - PostgreSQL client
- `bcrypt` - Password hashing
- `jsonwebtoken` - JWT tokens
- `zod` - Validation

### 5. Configure Environment

`.env` file is already configured with:
```bash
# Database
DB_HOST=localhost
DB_PORT=5432
DB_NAME=phishlogic
DB_USER=phishlogic
DB_PASSWORD=phishlogic_dev_password

# Auth
JWT_SECRET=phishlogic-dev-jwt-secret-key-change-in-production-min-32-chars
JWT_EXPIRES_IN=30d
```

### 6. Start Server

```bash
npm run dev
```

Expected output:
```
[INFO] Database connection initialized
[INFO] Fastify server configured
[INFO] Server started { host: '0.0.0.0', port: 3000, env: 'development' }
```

## Database Schema Overview

### Core Tables

1. **admin_users** - Admin accounts with JWT authentication
2. **api_keys** - User API keys with bcrypt hashing
3. **task_configs** - Per-task AI/Hybrid/Native configuration
4. **whitelist_entries** - Trusted emails/domains/URLs
5. **analyses** - Analysis results with cost tracking
6. **audit_logs** - All admin actions logged
7. **notification_configs** - Webhook/email/Slack alerts
8. **system_settings** - Key-value configuration store
9. **cost_summary** - (Placeholder for future aggregated costs)

### Multi-Tenant Ready Features

- All tables have `tenant_id UUID` column (nullable)
- Soft deletes with `deleted_at TIMESTAMPTZ`
- UUIDs for all IDs (globally unique)
- Request-scoped services (not singletons)

## Troubleshooting

### Database Connection Failed

```bash
# Check PostgreSQL is running
brew services list | grep postgresql
# or
sudo systemctl status postgresql

# Test connection
psql -U phishlogic -d phishlogic -h localhost -c "SELECT 1"
```

### Admin Login Failed

```bash
# Check admin user exists
psql -U phishlogic -d phishlogic -h localhost -c "SELECT username, email FROM admin_users"

# Re-run migration if needed
psql -U phishlogic -d phishlogic -h localhost < src/infrastructure/database/migrations/001_initial_schema.sql
```

### JWT Token Invalid

- JWT tokens expire after 30 days (configurable in `.env`)
- Login again to get a new token

### API Key Not Working

- API keys must start with `pl_`
- Pass as header: `X-API-Key: pl_...`
- Check if key is active: `SELECT is_active, expires_at FROM api_keys WHERE key_prefix = 'pl_...'`

## Security Notes

### Development vs Production

**Current (.env):**
- JWT secret is dev-only
- Database password is weak
- CORS allows localhost

**Before Production:**
1. Generate strong JWT secret (32+ chars)
2. Use strong database password
3. Enable SSL for database (`DB_SSL=true`)
4. Restrict CORS origins
5. Use environment-specific configs
6. Enable rate limiting
7. Set up monitoring/alerting

### Password Security

- Admin passwords hashed with bcrypt (cost factor: 10)
- API keys hashed with bcrypt
- Never log passwords or API keys

### API Security

- All admin routes require JWT token
- User routes accept API keys
- Audit logs track all admin actions
- Soft deletes preserve history
