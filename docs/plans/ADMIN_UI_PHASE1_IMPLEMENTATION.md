# PhishLogic Admin UI - Phase 1 Implementation Plan

## Executive Summary

Building a **single-tenant admin UI** with **multi-tenant-ready architecture**. This plan includes a robust authentication model that naturally evolves into full multi-tenancy.

**Timeline:** 3-4 weeks (17-20 days)
**Effort:** Solo developer or small team
**Migration Path:** 7-10 days to add full multi-tenancy when needed

---

## Context & Problem

**Current State:**
- All configuration via environment variables (requires server restart)
- No web UI for management
- No visibility into costs, logs, or analysis history
- No whitelist management UI
- No debug interface for troubleshooting

**Solution:**
React-based admin dashboard with 8 main features:
1. **Login Page** - Admin password + User API key authentication
2. **Task Configuration** - AI/Hybrid/Native per analyzer
3. **Whitelist Management** - Domains, emails, URLs, IPs
4. **Cost Analytics** - Track AI spending with charts
5. **Debug Interface** - Search analyses by ID
6. **Log Viewer** - Real-time structured logs
7. **Notification Settings** - Email, webhook, Slack config
8. **System Settings** - Budget, limits, retention policies
9. **API Key Management** - Generate/revoke keys (admin only)

## Database Schema (Multi-Tenant-Ready)

### Design Principles for Easy Migration

1. **Use UUIDs** - Globally unique, no conflicts when merging tenants
2. **Add tenant_id column** - Nullable now, required in Phase 2
3. **No foreign key to organizations table** - Doesn't exist yet
4. **Use TIMESTAMPTZ** - Timezone-aware for global deployments
5. **Include soft deletes** - deleted_at pattern
6. **Request-scoped services** - Not singletons

## Multi-Tenant Migration Path (Phase 2)

### When to Migrate

Migrate to multi-tenancy when you have:
- 25+ customers asking for multi-user support
- Demand from MSPs/resellers (need tenant isolation)
- Enterprise customers demanding SSO/RBAC
- Revenue >$500K/year

### Migration Steps (7-10 days)

#### Step 3: Update Services (Days 3-5)

```typescript
// Add tenant context injection middleware
export async function tenantContextMiddleware(
  request: FastifyRequest,
  reply: FastifyReply
): Promise<void> {
  // Extract tenant_id from JWT or API key
  const tenantId = request.user?.tenantId;

  if (!tenantId) {
    reply.status(401).send({ error: 'No tenant context' });
    return;
  }

  // Inject into request
  request.tenantId = tenantId;
}

// Update all services to use tenant_id from request
export function getWhitelistService(tenantId: string): WhitelistService {
  return new WhitelistService(tenantId); // Now required, not optional
}
```
